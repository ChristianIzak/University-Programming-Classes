# CSE220 Course Progress README

## Overview

This repository contains my work and practice from **CSE220**, where I have been learning core programming concepts in **C** with a focus on problem solving, memory, modular code design, and data organization.

So far, this course has helped me build a stronger understanding of how programs work at a lower level, especially through manual memory handling, pointer use, structured program design, and implementation of algorithms using C.

---

## Topics Learned So Far

### 1. Basic C Programming
I have learned the fundamentals of writing programs in C, including:
- variables and data types
- arithmetic and logical operations
- conditionals
- loops
- functions
- input/output with `printf` and related tools

These basics formed the foundation for the more advanced topics later in the course.

---

### 2. Modular Program Design
I have practiced splitting programs across multiple files using:
- `.c` source files
- `.h` header files

This helped me understand:
- function declarations vs. definitions
- how to organize larger programs
- how to reuse logic across files
- how separate program components work together

Examples of this appear in files involving utilities, matrix operations, max functions, maps, and scheduling work.

---

### 3. Arrays and Contiguous Memory
A major topic in this course has been arrays and how they are stored in memory.

I learned:
- how arrays are laid out contiguously
- how array names relate to pointers
- how to traverse arrays manually
- how to process 1D and 2D-style data

---

### 4. Pointer Arithmetic
One of the most important concepts I have worked on is pointer arithmetic.

So far I have practiced:
- moving through arrays with pointers
- dereferencing memory locations
- using pointers instead of `[]`
- understanding how pointer offsets relate to array positions
- treating flattened 2D data as 1D contiguous memory

---

### 5. Dynamic Memory Management
The course introduced manual memory control using:
- `malloc`
- `calloc`
- `realloc`
- `free`

Through memory management exercises, I learned:
- how to allocate memory at runtime
- how to resize allocated memory
- how to avoid memory leaks
- how improper memory handling can cause bugs or crashes
- how pointers interact with dynamically created arrays and structures

This was one of the most important shifts in the course because it moved beyond fixed-size data and into more flexible program design.

---

### 6. Matrices and Multi-Dimensional Thinking
I have also worked with matrices and matrix-style logic.

This included:
- representing matrix data
- accessing values by row and column
- understanding how 2D data can be stored in linear memory
- performing matrix-related operations

This helped reinforce both indexing logic and memory layout concepts.

---

### 7. Structs and Organized Data
Another major topic has been `struct`s.

I learned:
- how to define custom data types
- how to group related values together
- how to work with arrays of structures
- how to dynamically allocate structures
- how to pass structures to functions

This made programs more realistic and better organized, especially when modeling more complex information.

Examples include:
- copying and transforming data

These exercises helped me get more comfortable turning problem statements into working C code.

### 9. Debugging and Error Analysis
Throughout the course, I have also improved my debugging skills.

I have practiced:
- reading compiler errors
- fixing linker issues
- identifying logic bugs
- tracing segmentation faults and access violations

This has been a major part of learning C, since small mistakes in syntax, memory, or indexing can have large effects.
